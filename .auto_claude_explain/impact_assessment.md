## Impact Assessment

This change touches a **bash pipeline engine with file-based state**, not a database. Here are the entities, mapped to the requested framework:

### State Entities Read/Written

| Entity | Type | Read by | Written by | Change |
|---|---|---|---|---|
| `.auto_claude_state.json` | Mutable state file | All phases via `_state_get` | All phases via `_jq_update` | **New keys**: `.semantic.intent_map`, `.semantic.pattern_baseline`, `.semantic.pattern_conformance` |
| `.auto_claude_explain/` dir | Append-only audit trail | Quality-block phases, coverage tooling | New phases + existing seam | **New files**: `intent_map.json`, `pattern_baseline.json` |
| `OWNERSHIP_MANIFEST` (JSON file) | Read-only input (generated by worktree orchestrator) | `_load_ownership_manifest`, `_build_ownership_context` | `worktree_auto_claude` | **New consumer** — previously written but unused by auto_claude |
| `PHASES` array | In-memory dispatch table | Main loop | Declared once at top-level | **+2 entries**: `test_intent`, `pattern_baseline` |
| `OWNERSHIP_OWNS/READS/MUST_NOT_TOUCH` | Global shell vars | `_build_ownership_context`, `phase_plan`, `phase_implement` | `_load_ownership_manifest` | **New globals** |

### Dependency & Downstream Analysis

1. **No external consumers** — state files are pipeline-internal. No other system reads `.auto_claude_state.json` or the explain dir.
2. **No event contracts** — changes to state don't trigger external hooks. The pre-commit boundary fence hook is the closest analog; it reads `BOUNDARY_FENCE_FILE` (unchanged).
3. **Backward compatibility** — all new features are gated (`TDD_ENABLED`, `PATTERN_CONFORMANCE`, `ESCALATION_ENABLED`). Defaults are `false`/off, so existing `.auto_claude.conf` files produce identical behaviour.
4. **Blast radius** — confined to the pipeline's own working directory. worktree_auto_claude cherry-picks results via `git cherry-pick --no-verify`, so boundary hooks don't block the merge path.

**Risk**: The authoring conversation grows by ~2 phases of token budget. For large specs, plan context may degrade before `phase_implement`. This is noted in unverified assumptions and is the only non-trivial downstream concern.
